四、经验教训

✅ 成功经验

  • ✅ 按文本下载Excel确保数据格式正确，避免解析错误
  • ✅ 以「提交问卷时间」为唯一键，避免多用户协同重复
  • ✅ 二级标签标准化（前43行标准库），便于统计聚合
  • ✅ 纯SVG图表替代ECharts，确保GitHub Pages可靠渲染
  • ✅ 报告先发框架确认再出完整版，减少返工
  • ✅ GitHub Contents API直接推送，绕过git clone网络超时
  • ✅ 报告合集统一部署，外网永久可访问
  • ✅ KPI卡片式核心结论，用户原声表格化展示

⚠️ 踩坑记录

  • ⚠️ 问卷星按序号下载导致列格式错乱
  • ⚠️ 飞书API网络超时需重试机制
  • ⚠️ 浏览器自动化不稳定，需备选方案
  • ⚠️ 图片OSS链接有效期约90天，需定期更新
  • ⚠️ 飞书表格行数≠有效样本量，必须去空行
  • ⚠️ ECharts在GitHub Pages上不渲染，改用SVG
  • ⚠️ GitHub Pages文件名必须与链接完全一致
  • ⚠️ 飞书批量写入block容易报参数错误，需逐个写入

五、改进建议

5.1 产品层面

优先级问题建议
P0哨兵检测算法（漏报/误报）优化G-sensor阈值算法，区分碰撞/非碰撞场景；研发预计1.9.8版本优化
P0画面过曝（Bug[885465]）ISP调优，解决逆光进出高架桥洞场景过曝；熄火后ISP切换异常
P0固件升级进度条增加升级进度显示，避免用户以为设备卡死
P1APP白屏/闪退/开流失败优化APP稳定性，多机型兼容性测试
P1GPS定位精度提升首次定位速度和轨迹完整性
P1视频下载速度优化下载链路，产品已接受现状但需持续改进
P2双向通话音质提升4G网络下通话清晰度
P24G/Wi-Fi切换延迟优化网络切换逻辑，减少几十秒延迟

5.2 流程层面

🔄 流程优化

  • 建立日常数据自动同步机制，减少人工干预
  • 问题打标标准化，形成标签体系文档
  • 报告模板化，减少重复工作量
  • 巡检流程前置：每轮同步后立即验证数据完整性
  • 报告迭代版本管理：版本号+改动点+部署链接

🛠️ 工具优化

  • 问卷星API直连，避免浏览器自动化不稳定
  • 飞书多维表格自动同步，替代手动逐行写入
  • 报告自动生成流水线（数据→打标→图表→部署）
  • GitHub Pages部署脚本化，一键推送+验证
  • 图片OSS链接有效期监控，到期前自动更新
